# prompt

```md
以下のような仕様のGUIアプリを作りたいです．

GUI上に，

- Excelファイル選択ボタン（ボタンをクリックするとエクスプローラーでファイルを選択できる）
- 文字列入力ウィンドウ
- 検索範囲入力ウィンドウ（入力箇所は2か所あり，検索範囲の左上と右下と対応する）
- 保護範囲入力ウィンドウ（入力箇所は2か所あり，保護範囲の一番上の行と一番下の行と対応する）
- 実行ボタン

がある．

実行ボタンを押すと，

1. Excel内の保護範囲のデータが保持される（書式やセルの色等全て）
2. 検索範囲から文字列と完全一致するセルがある”行”が抽出される
3. 保護範囲のデータと抽出された全行が結合される．
4. 結合されたExcelファイルが出力される．

もう少し具体的なアルゴリズムを小さなExcelを例に説明すると，

- 保護範囲として1, 3が指定される
- 検索範囲としてB3, Q11が指定される
- 検索範囲内の対象文字列があるセルが特定される（例えば今回D4とK6にあったとする）
- そのセルが存在する行全体（検索範囲や保護範囲と関係なく）を抽出する（3行目，6行目全て）
- 保護範囲と抽出された行リストを結合する（そのため，保護範囲は少なくとも検索範囲と重複していたり，間に入ってはダメということ）（できあがるExcelファイルは保護範囲で指定された1行から3行目までのデータのしたに，4行目と6行目のデータが結合されたExcelとなる）

```
